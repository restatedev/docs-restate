## Install Restate Server & CLI

Choose your preferred installation method:

<Tabs>
  <Tab title="Homebrew">
    ```shell
    brew install restatedev/tap/restate-server restatedev/tap/restate
    ```
    Start the server:
    ```shell
    restate-server
    ```
    Find the CLI at:
    ```shell
    restate --help
    ```

  </Tab>
  <Tab title="Docker">
    Run the Restate Server:
    ```shell
    docker run --name restate_dev --rm \
      -p 8080:8080 -p 9070:9070 -p 9071:9071 \
      --add-host=host.docker.internal:host-gateway \
      docker.restate.dev/restatedev/restate:VAR::RESTATE_VERSION
    ```

    Run CLI commands:
    ```shell
    docker run -it --network=host \
      docker.restate.dev/restatedev/restate-cli:VAR::RESTATE_VERSION \
      invocations ls
    ```
    Replace `invocations ls` with any CLI subcommand.

  </Tab>

Look at the [CLI configuration docs](/operate/configuration/cli) or [Server configuration docs](/operate/configuration/server) for more configuration options.

<Note title="Telemetry">
  Restate Server collects anonymized telemetry about versions and uptime via [Scarf](https://about.scarf.sh).  
  We don't have access to your IP or any information about your cluster.
  To disable, set `DO_NOT_TRACK=1`.
</Note>

## Restate UI

The UI is bundled with the Restate Server and available at [http://localhost:9070](http://localhost:9070) when running locally. Use the UI to manage, debug, and configure your applications.


## Useful CLI Commands

With the CLI installed, try these commands:

- **Check which server you are connected to:**
  ```shell
  restate whoami
  ```

- **Register a new service deployment:**  
  (If using Docker, use `http://host.docker.internal:9080`)
  ```shell
  restate deployments register localhost:9080
  ```

- **List invocations:**  
  ```shell
  restate invocation list
  ```

- **Cancel a single or batch of invocations:**  
  Use `--kill` to force kill.  
  ```shell
  restate invocation cancel <INVOCATION_ID>
  # also works with <SERVICE>/<SERVICE_KEY>/<HANDLER> or a subset
  ```

- **Clear the K/V state of a Virtual Object or Workflow:**  

  ```shell
  restate kv clear <OBJECT_OR_WORKFLOW_NAME>
  restate kv clear <OBJECT_OR_WORKFLOW_NAME>/<SERVICE_KEY>
  ```

- **To wipe the server during local development**, remove the `restate-data` directory:
  This wipes all invocations, state, registered services, etc.
  ```shell
  rm -rf restate-data
  ```

- **Execute a SQL query on invocation or application state:**  
  See [SQL introspection docs](/operate/introspection?interface=psql#inspecting-invocations) for examples.  
  Use `--json` for JSON output.
  ```shell
  restate sql "query"
  ```
