<StateIntro/>

## List all state keys

To retrieve all keys the current Virtual Object has stored state for:

```rust
let keys: Vec<String> = ctx.get_keys().await?;
```

## Get state value

To read a value by key:

```rust
let my_string: Option<String> = ctx.get("my-string-key").await?;
let my_string = my_string.unwrap_or_else(|| "my-default".to_string());

let my_number: Option<i64> = ctx.get("my-number-key").await?;
let my_number = my_number.unwrap_or(0);
```

Returns `None` if the key does not exist.

## Set state value

To write or update a value:

```rust
ctx.set("my-key", "my-new-value".to_string());
```

## Clear state key

To delete a specific key:

```rust
ctx.clear("my-key");
```

## Clear all state keys

To remove all stored state for the current Virtual Object:

```rust
ctx.clear_all();
```

<StateLazy/>

## Lazy state loading per handler

You can opt individual handlers into lazy state loading using the `#[lazy_state]` attribute on the handler definition in the trait:

```rust
#[restate_sdk::object]
pub trait MyObject {
    // This handler fetches state on demand instead of eagerly on invocation
    #[lazy_state]
    async fn my_handler(name: String) -> Result<String, HandlerError>;
}
```

With `#[lazy_state]`, state is fetched from Restate Server only when `ctx.get(...)` is called, rather than being sent with the initial invocation. This reduces memory usage for handlers that do not access large state objects on every call.

Use this when:
- Your state is large and not always needed
- You want to reduce the initial request size
- Only a subset of handlers in an object need state
