`@restatedev/restate-sdk-gen` is an optional companion package that lets you write Restate handlers using JavaScript generators. It provides a composable DSL built around two concepts:

- **`Operation<T>`** — a lazy, one-shot description of work, built with `gen()`.
- **`Future<T>`** — an eager, memoized handle to an eventual value, returned by primitives like `run`, `sleep`, `awakeable`, and `spawn`.

All primitives (`run`, `sleep`, `all`, `select`, etc.) are free-standing functions you import directly. Inside a `gen(function*() { ... })` body they automatically resolve the active scheduler — no context parameter to thread through.

## Installation

```bash
npm install @restatedev/restate-sdk @restatedev/restate-sdk-gen
```

`@restatedev/restate-sdk` is a peer dependency.

### `sleep` — journaled timer

```typescript

```

### `awakeable` — journaled awakeable

```typescript

// Send `id` to an external system; it resolves `promise` when done.
const result = yield* promise;
```

### `channel` — in-memory channel for cooperative stop

A single-shot `Channel<T>` for signalling between concurrent routines within the same handler execution:

```typescript

// Producer:
yield* stop.send();
// Consumer:

```

## Combinators

Combinators work uniformly over journal-backed futures (`run`, `sleep`, `awakeable`) and routine-backed futures (`spawn`). The scheduler chooses the most efficient implementation.

### `all` — wait for all

```typescript {"CODE_LOAD::ts/src/develop/sdk_gen.ts#all"}
```

### `race` — first to settle

```typescript {"CODE_LOAD::ts/src/develop/sdk_gen.ts#race"}
```

The loser keeps running; its result is discarded.

### `any` — first to succeed

```typescript
const result = yield* any([
  run(() => callRegionA(), { name: "region-a" }),
  run(() => callRegionB(), { name: "region-b" }),
]);
```

Ignores rejections. Throws `AggregateError` only if every input rejects.

### `allSettled` — wait for all, regardless of failure

```typescript
const results = yield* allSettled([
  run(() => fetchA(), { name: "a" }),
  run(() => fetchB(), { name: "b" }),
]);
for (const r of results) {
  if (r.status === "fulfilled") console.log(r.value);
  else console.warn("failed:", r.reason);
}
```

## Cooperative cancellation

Use a `Channel<void>` and `select` to implement graceful stop:

```typescript {"CODE_LOAD::ts/src/develop/sdk_gen.ts#channel"}
```

`stop.receive` is the same `Future` on every access. After `yield* stop.send()`, every subsequent `select` with that branch takes it immediately. The stopped routine decides what to do — it can return partial results, run cleanup steps, or continue.

## Common pitfalls

**Do not reuse a generator instance.** `gen()` takes a factory function, not a generator. Pass `myWorkflow()` (a fresh `Operation`) to `spawn`, not the generator itself.

**Journal entry names must be deterministic.** The name is recorded in the journal and must match on replay. Avoid `Date.now()` or `Math.random()` in names. Use a counter variable instead.

**Do not call free functions outside an active fiber.** The free functions (`run`, `sleep`, `all`, etc.) read the active scheduler from a slot set by `execute()`. Calling them outside a `gen` body throws:
```
Error: restate-fluent: free-standing API called outside an active fiber.
```

## When to use `restate-sdk-gen`

The generator SDK provides an alternative authoring model, not additional capabilities. Use it when:
- Your workflow has complex concurrent patterns that are hard to express with `RestatePromise` combinators.
- You want to compose `Operation` values and pass them around like first-class values.
- You need multi-step sub-workflows with their own internal logic (via `spawn`).

The standard SDK (`@restatedev/restate-sdk`) with `ctx.run`, `ctx.sleep`, and `RestatePromise` combinators is sufficient for most workflows.
