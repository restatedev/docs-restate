---
title: Durability & Resilience
description: How Restate makes your code survive crashes and restarts
---

Restate tracks your code execution in a durable journal. When failures happen, Restate replays the journal to restore your application to exactly where it was - no duplicate work, no lost progress.

This durable execution model means you can write simple, sequential code without worrying about partial failures, network issues, or infrastructure problems. Your code will complete successfully even across crashes and restarts.

## How it works

1. **Journal tracking** - Restate logs each step of your code execution
2. **Automatic recovery** - On failure, Restate replays the journal to restore state  
3. **Exactly-once guarantees** - Operations are never duplicated during recovery
4. **Seamless continuation** - Your code picks up exactly where it left off
