Restate provides three service patterns optimized for different use cases. Choose the right type based on what your application needs to do.

## Handlers - The Functions That Do Work

Handlers are the functions inside your services that do the actual work. Each handler takes some input, processes it, and returns a result. You write normal functions, and Restate handles all the reliability, retries, and exactly-once execution automatically.

## Stateless Service – Fast Processing

Stateless services process requests without maintaining persistent state between invocations. They scale horizontally without coordination and are ideal for high-throughput data processing and business logic.

**Perfect for:**
- Data processing and transformations
- API integrations and external service calls
- Business logic and validation
- Mathematical computations
- Any task that doesn't need to remember previous requests

**Key characteristics:**
- No persistent state between requests
- Horizontal scaling without coordination
- High throughput and fast processing
- Automatic durability and retry capabilities
- Simple to develop, test, and deploy

## Virtual Object – Stateful Services

Virtual Objects are stateful services identified by a unique key. Each object maintains its own consistent key-value state that survives failures and restarts.

**Perfect for:**
- User accounts and profiles
- Shopping carts and sessions
- Game entities and player state
- Any business entity that needs to remember information between requests

**Key characteristics:**
- **Consistent state**: Built-in key-value storage per object
- **Single-writer guarantee**: Only one handler runs per key at a time
- **Automatic recovery**: State is preserved across failures
- **Per-key isolation**: Each object ID has completely separate state
- **No external database needed**: State is managed by Restate

Only one invocation can run at a time per object key, ensuring your state stays consistent without complex locking or coordination.

## Workflow Service – Long-Running Orchestration

Workflow services coordinate long-running processes that span minutes, hours, or days. They handle external signals, human interactions, and complex business logic while maintaining durability across any failures.

**Perfect for:**
- Approval processes requiring human input
- Multi-step transactions across different services
- User onboarding flows with email verification
- Order processing with payment, inventory, and shipping
- Any scenario requiring orchestration over extended periods

**Key characteristics:**
- **Long-running**: Execute for extended periods without consuming resources while waiting
- **Durable orchestration**: Coordinate multiple services reliably
- **Event-driven**: Respond to external signals and human input
- **Failure resilient**: Resume exactly where they left off after failures
- **Resource efficient**: Only consume resources while actively processing

## Choosing the Right Service Type

**Start with Stateless Services** for most business logic, data processing, and API integrations.

**Use Virtual Objects** when you need to maintain state for specific entities (users, orders, sessions).

**Use Workflows** for complex multi-step processes that involve coordination, human input, or long delays.

You can mix and match these service types within the same application - each serves different purposes and they work together seamlessly.
