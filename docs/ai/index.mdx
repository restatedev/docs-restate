Restate is the durable runtime your AI agents run on. It executes your agent loop with durable execution, so every LLM call, tool, and decision survives crashes, and it gives you one place to observe, steer, and operate every agent in production.

You don't rewrite your agent for Restate, and you don't adopt another framework. Restate is the layer underneath the one you already use, and the simplest way to take an agent from a demo to production.

## Where Restate fits in your stack

Restate is not another agent framework, and it is not a workflow engine you orchestrate from the outside. It is the durable runtime your agent code executes on:

- **Your agent logic** — a framework like OpenAI Agents, Vercel AI SDK, Google ADK, Pydantic AI, or LangChain, or a plain agent loop you write yourself.
- **Restate runtime** — runs that code with durable execution: journaling every step, recovering from failures, persisting state and sessions, queueing and coordinating calls, and exposing every running agent for inspection and control.
- **Your models, tools, and services** — LLMs, APIs, databases, and other agents.

<img src="/img/tour/agents/ai-app-layout.png" alt="Restate sits between your clients and your agent services, providing durable execution and orchestration" />

Restate replaces the resilience plumbing you would otherwise hand-roll: retries, idempotency, state stores, queues, schedulers, suspension, and observability. It does not replace your model provider or your agent framework.

## Why a runtime, not a library feature

Agent frameworks add durability by checkpointing state at boundaries. That recovers completed work, but it cannot supervise a crashed process, preserve retry state across restarts, coordinate concurrent writers to a session, or survive a redeploy mid-run. The component that guarantees a process runs to completion has to sit outside that process, which is why durable execution belongs in the runtime rather than the SDK.

Restate is that runtime. It is the same durable execution layer teams already use for payments and order workflows, applied to agents:

- **Checkpointing vs. durable execution** — see [why checkpointing is not production-grade durable execution](https://www.restate.dev/blog/why-checkpointing-is-not-production-grade-durable-execution).
- **Restate vs. Temporal** — Temporal makes a single agent loop durable; Restate makes the whole agentic system durable. See the [full comparison](https://restate.dev/vs/temporal).

From a single durable agent to multi-agent systems, harnesses, and platforms, it is the same runtime underneath.
